# Bilingual Support (FR/EN) — Design

## Approach
Lightweight i18n using React Context — no external library. Two languages: French (default) and English.

## Architecture

### Core: `src/lib/i18n.js`
- Translation dictionary: `{ fr: { ... }, en: { ... } }` with ~300 keys
- React context (`LocaleContext`) + provider (`LocaleProvider`)
- `useLocale()` hook returning `{ t, locale, setLocale }`
- `t('key')` looks up string, falls back to French if missing
- `t('key', { count: 3, name: 'x' })` for interpolation

### Language Detection & Persistence
- First visit: `navigator.language` (fr* -> fr, else -> en)
- Saved to `localStorage('locale')`
- Toggle in header (flag button next to theme toggle)
- `<html lang>` attribute updated on change

### Scope

**Translated (UI chrome):**
- All buttons, labels, headers, tooltips, placeholders, empty states
- Tab names, category names, column headers
- Date formatting locale (fr-CA / en-CA)
- Currency formatting locale
- Server error messages (codes on server, translated on client)
- Default role names for new users (language-aware defaults)
- Experience level display names

**Not translated (user data):**
- Role names, project names, phase names, milestone names, risk descriptions
- Existing saved rates/roles — no data migration

### Server Changes
- Return error code keys instead of French strings (e.g., `webhook_not_configured`)
- No other backend changes — language is frontend-only

### New Files
- `src/lib/i18n.js` — translations + context + hook
